    Основы PON

    Этапы диагностики:

    1. Проверить состояние ONU

    OLT-Starickogo-42-PON1#sh epon onu-in mac 80F7.A67E.B271
Interface EPON0/3 has registered 1 ONUs:
IntfName  VendorID  ModelID/Ext    MAC Address  Description  BindType    Status    Dereg Reason
--------  -------- ------------- --------------- ----------- -------- ------------ ------------
EPON0/3:28   CDT   110F/FD511G-E 80f7.a67e.b271      N/A     static   deregistered   wire-down

    Выводит информацию об ONU, что важно: IntfName = EPON0/3:28 - на OLT (коммутатор) есть физически порт №3, на нём доступно 64 виртуальных слота, на которые регистрируются ONU, в данном случае ONU зарегистрировалась на 28 место. Status = deregistered - ONU на OLT есть, но в статусе deregistered, т.е. связи с ней нет сейчас, есть ещё статус auto-configured, это значит что ONU зарегистрирована и получила конфиг от OLT, то есть должна работать нормально, registration это кратковременный статус во время загрузки ONU, в этот момент она получает конфиг от OLT, llid-admin-down это статус когда ONU по какой-то причине не может получить конфиг на OLT, будь то плохой приходящий сигнал или запрет регистрации со стороны OLT. Dereg Reason = wire-down - это значит что связь пропала по причине отсутствия оптического сигнала, вытащили оптику из ONU или обрыв к примеру, есть ещё статус power-off, значит ONU выключили из розетки.

    На основании этой инфы можно понять что делать дальше, к примеру, у статус power-off - стоит спросить абонента а горят ли вообще какие либо лампочки. Статус wire-down - не горит ли красная loss. Должна быть чёткая зависимость в голове от причины к последствиям, если что-то не сходится, то лучше уточнить или подумать ещё раз.

    2. Если ONU в статусе auto-configured, но всё равно не работает, нужно проверить связь с роутером:

    OLT-Starickogo-42-PON1#sh epon int epon0/3:28 onu port 1 state
    Hardware state is Link-Up
    Speed is 100Mbps
    Duplex is Full-Duplex

    Отображается link по витой паре от ONU до оборудования абонента, всё примитивно, link up - есть, down - нет, соответственно если down, то необходимо попросить абонента проверить кабель, отключить/подключить с двух сторон, если не помогло, проверить конфиг этой ONU командой sh run int epon0/3:28 на наличие строки epon onu port 1 ctc shutdown, она обозначает то что медный порт на ONU отключён, в такой ситуации его нужно включить.

    3. Если link по витой паре есть, нужно проверить приходит ли МАК роутера на эту ONU:

    OLT-Starickogo-42-PON1#sh mac ad dy in epon0/3:28
        Mac Address Table (Total 1)
------------------------------------------
Vlan    Mac Address       Type       Ports
----    -----------       ----       -----
1832    74da.8832.1077    DYNAMIC    epon0/3:28

    Если МАК есть - а сессии всё ещё нет, нужно сверить Vlan, тут МАК роутера во vlan 1832, проверить какой vlan на epon0/3 при помощи команды sh run int epon0/3, в самом низу будет строка switchport vlan allowed и там прописан vlan, если сходится с тем куда пришёл МАК роутера, то должно быть всё в порядке, скорее всего нужно проверять настройки на роутере, может не выбрано DHCP в типе подключения
